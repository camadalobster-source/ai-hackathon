01 —

CLIP 技術說明

一句話理解 CLIP

CLIP 不是分類系統，也不是生成系統，而是一個語意橋接器——它讓「文字」與「圖片」能在同一個向量空間中比對相似度。輸入文字，找出最相符的圖片；輸入圖片，找出最相符的文字描述。

技術運作原理

1

對比學習（Contrastive Learning）

CLIP 同時訓練「文字編碼器」與「圖片編碼器」。語意相近的圖文配對在向量空間中被拉近，不相關的被推遠。這是 Zero-shot 能力的根本來源。

2

訓練資料規模（Notebook LM 未提及）

訓練資料為 4 億組圖文配對（WebImageText 資料集），全部來自網路爬取。訓練資料集本身是 OpenAI 私有的，工程師取得的是模型權重與程式碼，而非原始訓練資料。

3

Zero-shot：開箱即用，無需重訓

CLIP 不需要預先設定分類桶，也不需要重新訓練。給它一張圖片加上幾個自然語言描述選項，它計算相似度並回傳最匹配的結果，能即時處理從未見過的視覺概念。

解決了哪些過往成本過高的痛點

痛點一

省下天價標記成本

ImageNet 動用 25,000 名工人標記 1,400 萬張圖。CLIP 直接從網路現成圖文資料學習，大幅降低人力成本。

痛點二

打破「一模型一任務」

傳統模型新增類別需重新收集資料並微調。CLIP 只要用自然語言告訴它新分類名稱，即可立刻執行。

痛點三

解決「實驗室好、現實差」

傳統模型針對測試集死背，遇到素描或不同角度就失敗。CLIP 在真實多變環境下展現更高穩定性。

痛點四

跨模態搜尋成為可能

可用文字搜圖、用圖片搜圖，不需人工標註的 metadata，特別適合缺乏完整標記的大型影像素材庫。

Token 上限（重要技術限制）

有效文字輸入：約 10–13 個中文字

CLIP 文字輸入上限為 77 個 token，但實際有效長度僅約 20 個 token（約 10–13 個中文字）。超過後影響力快速下降。搜尋描述要精簡：「工廠火災濃煙」比長句更有效。

升級路徑：Long-CLIP（2024）可大幅提升有效長度，在長文字描述的圖片檢索上比原始 CLIP 提升約 20%，且可直接替換使用。

已知技術限制（使用前必讀）

OCR 能力不穩定

圖片文字辨識準確率約 50%。適合輔助初篩，不適合作為精確文字提取主力工具。

無法計數與空間判斷

「幾個人」、「物件距離」等任務表現接近隨機猜測。不適用需要精確數量的報導場景。

細微差異辨識能力弱

特定汽車型號、飛機機型、品牌招牌等細節辨識，不如任務專用模型。

事件識別能力不足

「這是山陀兒颱風的畫面」這類需識別特定歷史事件的任務，CLIP 做不到。應交給 OpenSearch。

偏見與道德風險

若分類選項含強烈負面詞彙，有 ~32.3% 機率將 0–20 歲年輕人錯誤歸類。用於人物標記需特別謹慎。

無法生成文字描述

CLIP 是「選擇題」模型，無法生成任何文字。需要 alt text 等功能，必須串接 Vision-Language Model（如 GPT-4V）。

02 —

CLIP 在新聞產業的應用

應用判斷大原則

✓ 適合：「找情境、找概念、找畫面感」
✗ 不適合：「算人數、認特定人物、辨識精細型號、識別特定事件、生成任何文字」

四個核心應用場景

場景一 ✓ 最成熟

歷史圖庫語意搜尋

記者可輸入「電視攝影棚的照片」或「飛機與人群」，直接找出對應新聞照片，無需人工標籤。GDELT Project 已驗證此方法可搜尋超過 19 億張電視新聞畫面。

已有媒體驗證

場景二 △ 輔助初篩

UGC 事實查核輔助

地理定位能力可協助比對照片背景是否與聲稱地點相符。語意 OCR 可輔助初步判斷截圖內容。準確率有限，僅適合初篩。

準確率有限

場景三 △ 需額外處理

影音素材快速掃描

對影片每幀提取向量，可搜尋特定動作或場景出現的時間點。需先進行影片截幀處理，屬升級版功能，本次提案範圍外。

影片為未來升級方向

場景四 △ 高風險

社群惡意圖文初步偵測

可在無需真實標籤的情況下偵測仇恨迷因。但偏見風險高，人工審核不可省略。

偏見風險高
